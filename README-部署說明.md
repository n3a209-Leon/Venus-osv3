# LIMU 教師專用小工具 v12

## 部署

請保留原有目錄結構，將這個資料夾內的所有檔案一起上傳到網站根目錄。不要只上傳 `index.html` 與 `sw.js`；`version.json`、圖片、PWA 圖示與本機 ReactDOM 都是必要檔案。

部署後第一次開啟時請保持網路連線。v12 會先核對 `index.html`、`sw.js` 與 `version.json` 的建置識別，全部一致且必要資源快取完成後才顯示「立即更新」。按下後會一次切換並自動重新載入；若偵測到混版，App 會主動警告重新上傳完整部署包。

`firestore.rules` 是 Firebase Firestore 規則參考檔，不需放進網頁主機；只有在 Firebase 規則尚未部署時才需要到 Firebase Console 更新。

## 本版重點

- 新增完全免費的公開學校行事曆：不使用 Cloud Functions、不需要 Blaze、不嵌入 Google API 金鑰，也不要求私人 Google 行事曆授權。
- 可輸入學校網站自動偵測公開 Google Calendar、`.ics`、CSV、JSON 與網頁 JSON-LD 事件。
- 已內建大甲國小目前公開校務行事曆的免費辨識備援；即使首頁本身不允許跨網域讀取，仍會嘗試讀取其公開 Google 行事曆。
- 若學校網站禁止跨網域讀取，畫面會明確要求改貼公開行事曆直接連結，或匯入已下載的 `.ics`／CSV／JSON；不會使用第三方代理。
- 所有外部事件先進入勾選清單，支援全選、全不選與逐筆決定，確認後才加入目前班級。
- 公開事件與既有手動行事曆分開保存；不改寫舊資料，不影響每日作業、出勤、輔導或週計畫。
- 當日作業紀錄上方可展開事件完整內容、標記 `★重要`，並以「只留重要」一鍵隱藏一般事件。
- 刪除公開事件後會記入「已排除」，下一次匯入不會自動加回；可在匯入畫面恢復誤刪事件。
- 支援同一天多筆公開事件、處室／來源標示、重複行程展開與跨日事件。
- 公開事件按班級隔離，並沿用既有 Firestore revision 衝突保護與自動重試；Firebase 專案和規則不需更換。
- 完整封存與加密備份會包含公開事件、重要標記、來源設定與排除清單。
- 學年度轉換目標固定為「目前班級學年度＋1」，116、117、118 等後續年度可持續使用。
- 同一來源班級不能重複建立相同學年度；同校、同班名、同學年度也會阻止重複。
- 手動新增班級若撞到既有資料，會直接切換到原班級，不再產生重複副本。
- 上下學期開始日與匯入行事曆會核對學年度範圍，避免日期誤填到前一年或後一年。
- 轉換紀錄保留來源學年度與轉換時間，方便日後確認資料來源。
- 班級設定、組別版本、幹部及行事曆全部加入 revision 交易保護，跨裝置衝突不再由最後寫入者直接覆蓋。
- 設定衝突整合至既有「同步衝突」中心，可選擇使用此裝置或雲端版本。
- Firebase 暫時失敗時，會依 5 秒、15 秒、45 秒、2 分鐘、5 分鐘延遲自動重試；衝突資料不會無限重送。
- 設定頁顯示 App 版本、建置日期、快取版本與資料格式版本。
- 通用學年度轉換助手：先檢查舊學年完整封存，再建立下一學年度班級、設定開學日並匯入新行事曆。
- 轉換時可複製組別名稱與幹部職稱，但不會複製學生座號或幹部人選。
- 行事曆改為按班級獨立保存，並保留舊版全域行事曆的自動遷移相容性。
- Service Worker 改為使用者確認後才啟用，避免新舊資源短暫混用。
- 週計畫與輔導紀錄可離線新增、編輯與刪除，重新連線後補同步。
- 每日資料、週計畫與正式輔導都使用交易式版本檢查；偵測到另一台裝置已修改時不直接覆蓋。
- 功能選單新增「同步衝突」中心，可選擇本機、雲端、合併週計畫或讓兩筆輔導都保留。
- 完整備份包含每日紀錄、日輔導、週計畫、正式輔導、分組、幹部、設定與行事曆，並附 SHA-256 校驗。
- 下載備份時可選擇 AES-256-GCM 密碼保護；仍相容一般 JSON 與舊版備份。
- 雲端保留最近 3 份完整封存；新封存成功後才更新目前版本，並可安全清理舊分段。
- 本機網站儲存空間超過 80% 時才顯示提醒，平常不增加常駐儀表板。
- 行事曆過期或即將到期時主畫面會提醒，匯入時會驗證格式。
- 支援 PWA manifest、192／512 圖示、離線 App Shell、鍵盤焦點與系統減少動態效果設定。
- 大型圖片已移出 HTML，`index.html` 體積約減少 48%。

## 建議操作

重要學期資料請定期到「學年歷史檔案室」下載密碼保護備份，並把密碼存放在可信任的密碼管理工具。密碼遺失後無法解密。第一次從其他裝置匯入後，請確認班級、週計畫與輔導紀錄皆可見。

公開行事曆單次來源上限為 5 MB、最多保留 3,000 筆事件。學校網站只能在對方允許瀏覽器跨網域讀取時自動取得；遇到限制時，下載公開檔案再匯入是完全免費且最穩定的備援方式。
